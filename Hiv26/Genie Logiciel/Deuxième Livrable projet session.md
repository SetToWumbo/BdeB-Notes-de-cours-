#### Diagramme de cas d'utilisation (use case)
Choisir un modèle de Use Case et l'utiliser pour tous les cas.

Pour l'IEEE:
Voir le document fourni dans le cours
- Donner ID au use case
- Titre au use case(déroulement normal-nominal), verbe correspondant a ce que l'acteur veut faire
- Acteurs
- But(Qu'est-ce que l'acteur veut accomplit en utilisant cette fonction)
- Sommaire (deux phrases sur l'ordre du déroulement du use case)
- Références (d'où vient le use case, doit indiquer l'origine du spec, indiquer le document contenant le spec dans la section références du document EX: énoncé du projet)
- Priorité ()
- Préconditions: liste des conditions qui doivent être vraies pour que le use case s'applique
- PostConditions: garantie a la sortie. Si use case s'est bien déroulé, qu'est-ce qu'on peut garantir?
- Exigences complémentaires: Quelles sont les validations que le système doit faire, etc..
